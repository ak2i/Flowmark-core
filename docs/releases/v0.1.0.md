# FlowMark v0.1.0 Release Notes

Date: 2026-02-09

## Summary

FlowMark v0.1.0 delivers the first public CLI for validating FlowMark documents. It includes a minimal parser, validator, and Docker image, plus starter samples and templates.

## Highlights

- CLI commands: `validate`, `parse`, `lint`
- Strict vs lenient mode with expected/ unexpected item handling
- Coverage validation via registry (`expected_items`)
- Docker image published for Node-free usage

## CLI

- `flowmark validate`: full validation (MUST rules) + warnings
- `flowmark parse`: parse result as JSON (no validation)
- `flowmark lint`: SHOULD-focused warnings

## Validation Rules Implemented

- Header cardinality and required fields
- Item cardinality, id uniqueness, and status validation
- Registry coverage checks and duplicate IDs
- Unknown keys warnings
- YAML parse errors are surfaced as `E_YAML_PARSE`
- If `E_YAML_PARSE` occurs, derived errors/warnings are suppressed

## Samples

- Minimal conformant example
- Spec-conformant example
- AI authoring template
- Error cases for duplicate header, missing items, bad status, coverage mismatch, invalid YAML, unexpected items

## Docker

Image: `ghcr.io/centimanidevteam/flowmark-core-cli`

## Notes

- This release focuses on the CLI and validation behavior.
- Library APIs are intentionally deferred.
